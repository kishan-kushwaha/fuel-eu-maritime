# AI Agent Workflow Log

## Agents Used
- ChatGPT for architecture planning, scaffolding, and documentation drafting
- GitHub Copilot for inline completion while expanding repetitive TypeScript code
- Thunder Client for API validation with AI-guided request/response checks



## Development Workflow

The project was primarily implemented manually, following the assignment requirements.
AI tools were occasionally used to speed up debugging, confirm architectural decisions, and generate small helper snippets.

The overall implementation process followed these steps:

- Designed the project structure using a hexagonal architecture pattern.

- Implemented the PostgreSQL schema for routes, compliance balances, and pooling.

- Built backend services and REST APIs using Node.js and TypeScript.

- Developed the React dashboard with separate tabs for routes, comparison, banking, and pooling.

- Integrated frontend components with backend APIs.

- Used AI tools selectively for troubleshooting and minor improvements.

## Example Interactions With AI Tools
# Example 1 – Architecture Clarification

## Prompt ##

- Suggest a clean project structure for a Node.js + TypeScript backend following hexagonal architecture.

## Result ##

The response helped validate the use of separate folders for:

- domain entities

- application use-cases

- ports (repository interfaces)

- adapters (controllers and database implementations)

This structure was then implemented manually in the project.

## Example 2 – Debugging Assistance ##

## Prompt ##

The baseline route update should ensure only one route is marked as baseline. How should this be handled safely in PostgreSQL?

# Result

The suggestion recommended wrapping the update in a database transaction.
The final implementation uses a transaction to first reset the existing baseline and then set the new one.

## Example 3 – Frontend Data Flow

# Prompt

How should React components fetch route comparison data from the backend API?

# Result

The suggestion helped confirm the pattern of using a small API client layer with hooks for fetching data from REST endpoints.

## Validation and Manual Review

All code suggestions generated by AI tools were manually reviewed before being included.

Specific validation steps included:

- verifying API routes match the assignment specification

- confirming the baseline comparison logic and FuelEU target value

- checking that only one baseline route exists at a time

- validating database queries against the schema

- ensuring React components remain separate from domain logic

- Any unclear or overly complex generated code was simplified manually.

## Observations

AI tools were most useful for:

- debugging errors

- validating architectural ideas

- generating small helper snippets

- improving documentation structure

- However, domain-specific logic (such as compliance           calculations and pooling behavior) required manual implementation and verification.
